Aegis

HTTP proxy guardrail for AI agent penetration testing · written in Rust

What it is

Aegis is a Rust forward proxy that wraps AI agents (Claude Code, Codex CLI, or any custom tool) during offensive security tasks. It intercepts HTTP/HTTPS traffic and shell commands, blocking destructive actions before they reach client infrastructure while letting legitimate offensive operations pass through.

Why it exists

Autonomous AI agents running with full permissions (--dangerously-skip-permissions, --full-auto) can cause real damage during pentesting engagements. A misinterpreted prompt can turn SELECT * FROM users into DROP TABLE users. Aegis sits between the agent and the network, enforcing scope boundaries and blocking destructive operations in real time.

What it does

  • Scope enforcement · intercepts every HTTP/HTTPS request and shell command; blocks anything outside the engagement’s authorized scope.
  • Kernel-isolated sandboxing · macOS Seatbelt and Linux Landlock to confine the agent at the OS level, not just the proxy layer.
  • Proxy-based secret injection · API keys are injected by the proxy on egress, so they never touch the agent process or its prompt context.
  • Policy system · declarative policies and presets, plus custom middlewares for engagement-specific rules.
  • Audit logs · full request/response and shell-command record for after-action review and client transparency.
  • Anti-evasion · blocks common bypass tricks an agent might attempt under prompt influence.
  • Web & TUI dashboards · live observation of agent behaviour during the engagement.
  • SDK / API · integration endpoints for Python and TypeScript agent frameworks.

Where it fits

The core idea is preventing accidental damage from agents running real offensive security tasks: scope drift, destructive commands, credential leakage, or out-of-bound requests. Aegis sits in front of the agent and blocks those specifically, while letting the intended offensive operations go through.
