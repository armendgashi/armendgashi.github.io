insecure_agentic_api

Small, intentionally insecure FastAPI examples showing common failure modes in agentic / tool-using LLM APIs

The idea

When teaching people how agentic LLM systems fail, it helps to have tiny, self-contained reproductions you can run and prod. insecure_agentic_api is exactly that: a handful of short, deliberately vulnerable scripts, each focused on a single failure mode, so the class of bug is obvious without wading through a full application.

The repo is used as hands-on material in workshops and for research on Special Token Injection, LLM API misconfigurations, and agent-level role injection.

What’s in it

Four FastAPI / Python scripts, one vulnerability pattern each.

  • parameter_tampering.py · a full chat-completions wrapper that forwards arbitrary request parameters to the OpenAI client. Clients can override server-side intent: pick a different model, disable safety-relevant knobs, inject tools, or change tool_choice / response_format. A textbook case of “proxy more than you meant to.”
  • raw_sti.py · a local Special Token Injection demo using Qwen2.5-1.5B-Instruct and a tool that executes SQL against a SQLite database. Shows how untrusted input can reshape the model’s structural tokens and steer tool-call arguments into dangerous SQL.
  • role_injection.py · a multi-agent pipeline (code-gen agent → triage agent → language-specific executor). Demonstrates prompt/role injection across the agent graph, including bypass of the “no OS libraries” scope.
  • unbounded_consumption.py · a simplified chat API with HTTP tool execution that can be steered into unbounded tool usage and resource consumption (cost-blowup, loop-until-cap behavior).

How to run it

Requirements: Python 3.10+, an OpenAI API key for the FastAPI examples, plus torch / transformers for raw_sti.py.

pip install fastapi uvicorn openai httpx pydantic
pip install torch transformers  # for raw_sti.py
export OPENAI_API_KEY="your-key"

uvicorn parameter_tampering:app --reload
uvicorn role_injection:app --reload
uvicorn unbounded_consumption:app --reload

Then POST to the exposed endpoints (/chat, /ask, etc.) and try to break them.

Why it exists

Most public examples of agentic API code are either a real product (too much noise to see the bug) or a sanitized snippet (too little context to reproduce). These four scripts sit in between: big enough to behave like the real thing, small enough to reason about the attack surface end-to-end. They pair with the write-ups on Special Token Injection and LLM API Misconfigurations.
