Agentic Offensive Security Active R&D

Multi-agent LLM systems for source-code review, validation, and exploitation

Premise

Most “AI for security” tooling sits on the defensive side: anomaly detection, log triage, alert summarization. The offensive side has been slower to mature because the work is harder to formalize: it requires multi-step reasoning, code understanding, real validation (not just plausible-looking output), and the discipline to discard false positives instead of inflating findings.

This research builds and operates multi-agent systems against that gap, with explicit roles for hypothesis generation, validation, exploitation, and reporting.

Capabilities

  • Source-code vulnerability identification · semantic analysis across files and language boundaries, not single-line pattern matching.
  • False-positive validation on Semgrep findings · agent traces taint, evaluates reachability, and discards findings that are unexploitable in context.
  • Proof-of-concept generation · the agent constructs and executes a working PoC, instead of asserting impact.
  • Report generation · structured technical and executive output, suitable for direct delivery into client-facing reports.
  • Web & API penetration testing assistance · reasoning over request/response flows, auth state, and business logic.

Productized as SakiX

The capabilities above are integrated into SakiX, an internal agentic system used inside Sentry Security on real client engagements. SakiX serves as both the production target and the experimental harness for ongoing research.

Adjacent work

  • Special Token Injection (STI) · offensive AI capability that also exposes attack surface back at the LLM stack itself.
  • EX MACHINA · the workshop where these methods are taught hands-on.
